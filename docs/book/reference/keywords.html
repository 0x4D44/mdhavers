mdhavers - A Scots Programming Language

Keywords Reference

Complete reference of all mdhavers keywords with their Scots meanings and usage.

Variable Declaration

ken

Meaning: “To know” or “to understand” Usage: Declare variables

ken name = "Hamish"
ken age = 30
ken is_scottish = aye

Conditionals

gin

Meaning: “If” Usage: Start a conditional block

gin age >= 18 {
    blether "Ye're an adult!"
}

ither

Meaning: “Other” or “else” Usage: Alternative branch in conditionals

gin score >= 70 {
    blether "Ye passed!"
} ither {
    blether "Try again."
}

than

Meaning: “Then” Usage: Ternary expressions

ken result = gin x > 0 than "positive" ither "not positive"

Loops

whiles

Meaning: “While” Usage: While loop

ken count = 0
whiles count < 5 {
    blether count
    count = count + 1
}

fer

Meaning: “For” Usage: For loop

fer i in 1..10 {
    blether i
}

fer item in my_list {
    blether item
}

in

Meaning: “In” Usage: Iteration in for loops

fer x in collection {
    # ...
}

brak

Meaning: “Break” Usage: Exit a loop early

fer i in 1..100 {
    gin i > 10 {
        brak
    }
}

haud

Meaning: “Hold” (as in “hold on”) Usage: Continue to next iteration

fer i in 1..10 {
    gin i % 2 == 0 {
        haud  # Skip even numbers
    }
    blether i
}

Functions

dae

Meaning: “To do” Usage: Define a function

dae greet(name) {
    blether f"Hullo, {name}!"
}

gie

Meaning: “To give” Usage: Return a value from a function

dae add(a, b) {
    gie a + b
}

Boolean Values

aye

Meaning: “Yes” Usage: Boolean true

ken is_ready = aye

nae

Meaning: “No” or “not” Usage: Boolean false, or logical NOT

ken is_empty = nae
ken is_invalid = nae is_valid

naething

Meaning: “Nothing” Usage: Null value

ken result = naething

Logical Operators

an

Meaning: “And” Usage: Logical AND

gin age >= 18 an has_license {
    blether "Can drive"
}

or

Meaning: “Or” Usage: Logical OR

gin is_admin or is_owner {
    blether "Has access"
}

Input/Output

blether

Meaning: “To chat” or “to talk” Usage: Print output

blether "Hullo, World!"
blether 42
blether my_list

speir

Meaning: “To ask” Usage: Get user input

ken name = speir "Whit's yer name? "

Classes

kin

Meaning: “Family” or “type” Usage: Define a class

kin Person {
    dae init(name) {
        masel.name = name
    }
}

fae

Meaning: “From” Usage: Class inheritance

kin Dog fae Animal {
    dae speak() {
        blether "Woof!"
    }
}

masel

Meaning: “Myself” Usage: Reference to current instance (like this or self)

kin Counter {
    dae init() {
        masel.count = 0
    }

    dae increment() {
        masel.count = masel.count + 1
    }
}

thing

Meaning: “Thing” Usage: Define a simple struct (record type)

thing Point {
    x,
    y
}

Modules

fetch

Meaning: “Fetch” (to get) Usage: Import a module

fetch "lib/helpers"

tae

Meaning: “To” Usage: Import module with alias

fetch "lib/math" tae m
blether m["square"](5)

Error Handling

hae_a_bash

Meaning: “Have a bash” (try something) Usage: Try block

hae_a_bash {
    ken result = risky_operation()
} gin_it_gangs_wrang err {
    blether f"Error: {err}"
}

gin_it_gangs_wrang

Meaning: “If it goes wrong” Usage: Catch block

hae_a_bash {
    # risky code
} gin_it_gangs_wrang error {
    # handle error
}

Pattern Matching

keek

Meaning: “Peek” or “look” Usage: Match expression

keek day {
    whan 1 -> { blether "Monday" }
    whan 2 -> { blether "Tuesday" }
    whan _ -> { blether "Other day" }
}

whan

Meaning: “When” Usage: Match case

keek value {
    whan 0 -> { gie "zero" }
    whan x -> { gie f"got {x}" }
}

Assertions

mak_siccar

Meaning: “Make sure” (famously said by Robert the Bruce!) Usage: Assert a condition

mak_siccar x > 0, "x must be positive"
mak_siccar len(list) > 0

Type Checking

is

Meaning: “Is” Usage: Type checking (with is_a function)

gin is_a(x, "integer") {
    blether "It's a number!"
}
